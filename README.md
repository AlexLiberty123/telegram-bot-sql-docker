Телеграм-бот для аналитики видео-статистики (просмотры, лайки, комментарии) с использованием естественного языка. Бот преобразует вопросы пользователя в SQL-запросы к PostgreSQL, используя LLM (**arcee-ai/trinity-large-preview:free** через OpenRouter).

## Функционал
- **Естественный язык:** Понимает вопросы вроде "Сколько всего просмотров?", "Какой прирост лайков был 26 ноября?".
- **Устойчивость:** Игнорирует оффтоп, приветствия, попытки узнать ID или текстовые поля.
- **Точность:** Возвращает строго числовые значения.
- **Архитектура:** Docker + С + PostgreSQL.

## Установка и запуск
Проект полностью докеризирован. Требуются установленные **Docker** и **Docker Compose**.

## 1. Клонирование репозитория
```
git clone https://github.com/AlexLiberty123/telegram-bot-sql-docker.git
cd telegram-bot-sql-docker
```

## 2. Настройка окружения
В корне проекта есть файл .env.example. Создайте на его основе файл .env:

Linux/Mac:
```
cp .env.example .env
```

Windows:
Создайте файл с именем .env и скопируйте туда содержимое из .env.example.

Обязательно откройте файл .env и заполните свои данные:

TG_TOKEN: Ваш токен от @BotFather.
OPENROUTER_KEY: Ваш API ключ OpenRouter.
Остальные настройки БД (DB_HOST, DB_USER и т.д.) можно оставить по умолчанию для Docker.

## 3. Запуск
```
docker-compose up --build
```
Что произойдет после запуска:

Развернется контейнер с базой данных PostgreSQL.
Скрипт bot/db_init.py автоматически создаст таблицы и импортирует данные из файла videos.json в базу.
Запустится бот и будет готов отвечать на вопросы.

## Архитектура и Промпт
Схема данных для LLM
В системном промпте мы описываем структуру базы, разделяя её на две логические части, чтобы модель понимала контекст:

Таблица videos: Используется для запросов "Всего" (Total stats / Lifetime).
Таблица snapshots: Используется для запросов "За дату", "Прирост" или "Динамика" (Growth / History).
Системный промпт
Используется строгий промпт (Role: Strict SQL Generator) для защиты от галлюцинаций:

Правила:
Если вопрос подразумевает числовой ответ -> вернуть SQL.
Если вопрос про ID, текст, имя или оффтоп -> вернуть ключевое слово IGNORE.
Автоматически исправлять опечатки пользователя.
Формат ответа: Только чистый SQL код без Markdown и пояснений.
